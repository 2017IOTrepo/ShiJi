本项目经历过一次重构，将原来的`kratos`微服务框架改为了使用`Gin`框架~~(咋还越用越垃了)~~

主要原因是个人后面比赛的时候发现臃肿繁琐的微服务框架拖累了个人开发效率(这种设计就跟中台方案一样大抵只适用与大型项目)，所以转而使用了轻便的`Gin`框架(这就是为什么是apiv2)
